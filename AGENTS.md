# AGENTS.md — Garmin Visual Insight Bot (v1)

## Core product rules
1. Пользовательские тексты по умолчанию только на русском языке.
2. Запрещён медицинский тон: никаких диагнозов, предупреждений и «лечебных» формулировок.
3. Интерпретация данных только как «ритм / режим / состояние дня».
4. Минимализм в сообщениях: коротко, по делу, без перегруза эмодзи.
5. Без «инстаграм-мотивации» и пафосных формулировок.
6. Юмор допустим редко, сухо и ненавязчиво: максимум 2 раза в неделю.
7. При fallback (нет данных за день) юмор не используется.

## Engineering constraints
8. Детерминированная логика предпочтительнее ML.
9. Минимум зависимостей, без усложнения архитектуры.
10. Все фичи должны корректно работать при частично отсутствующих полях Garmin.
11. Инкрементальные изменения: не ломать существующие рабочие потоки.
12. Добавлять тесты по мере целесообразности, без оверинжиниринга.
13. Ошибки и edge-cases должны иметь надёжный fallback + логирование.

## Visual & color system
14. Визуалы только процедурные (Pillow), без каталогов картинок и без scraping.
15. Изображение дня: PNG 1080x1080, детерминированный seed, анти-повторы (recently used).
16. Генерация кэшируется; одинаковый вход не должен пересоздавать изображение.
17. Система недельного цвета обязательна: выбор 1 раза на ISO-неделю, с hex + name_ru + rarity_level.
18. Истории цвета короткие (2–3 предложения), из внутреннего словаря, без web scraping.

## Product scope v1
19. Daily card: краткий insight + визуал + опционально редкий юмор.
20. Fallback для дня без данных обязателен.
21. Weekly report: контрасты и диапазоны, не более 4–6 метрик.
22. Обратная связь: кнопки ✅ / 🤷 / ❌, хранение голосов и недельная статистика.
23. Без платных API: решение должно быть автономным.
