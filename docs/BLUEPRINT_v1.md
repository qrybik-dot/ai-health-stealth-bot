# Blueprint v1 — Garmin Visual Insight Bot

## Mission
Собрать стабильную v1-версию бота с минимальной сложностью:
- процедурный ежедневный визуал (без каталогов изображений и без scraping);
- система недельной цветовой идентичности (полный спектр + иногда редкие исторические названия);
- daily card: краткий инсайт + изображение + опционально редкий юмор;
- fallback при отсутствии Garmin-данных за сегодня;
- weekly report: контраст и диапазоны, максимум 4–6 метрик;
- 3-кнопочный feedback («✅ / 🤷 / ❌») с хранением и недельной статистикой.

## Non-negotiable product rules
- Пользовательский текст по умолчанию на русском языке.
- Запрещён медицинский тон: без диагнозов, предупреждений и «медицинских» трактовок.
- Сигналы интерпретируются как «ритм / режим», не как лечение.
- Без «Instagram-motivation» стилистики.
- Юмор — сухой, редкий, ненавязчивый: максимум 2 раза в неделю.
- В fallback-сценарии при отсутствии данных юмор не используется.
- Минимализм: короткие тексты, без визуального шума и emoji-overload.

## Engineering rules
- Предпочитать детерминированную логику вместо ML.
- Держать зависимости минимальными.
- Обеспечить работу при частично/полностью отсутствующих полях Garmin.
- Не ломать существующие рабочие потоки; изменения инкрементальны.
- Добавлять тесты там, где это оправдано, без переусложнения.
- Добавить надёжную обработку ошибок и логирование.
- Кэшировать сгенерированные изображения (без лишней регенерации).
- Реализовать анти-повтор визуалов через детерминированные seed и список recently used.

## Visual system (procedural)
- Генерация PNG 1080x1080 кодом (предпочтительно Pillow).
- Состав: градиентный фон от weekly color + 1–3 абстрактные формы + лёгкое зерно/шум.
- Вариативность композиции определяется mode_tag:
  - recovery → мягче и спокойнее;
  - push → резче и контрастнее;
  - balance → нейтрально и чисто.
- Seed = hash(chat_id, date, mode_tag, week_id).
- Для одинаковых входов результат идентичен.
- При ошибке генерации — fallback на простой градиент.

## Color system
- Цвет недели выбирается 1 раз на ISO-неделю.
- Подбор на основе HSL/LAB с фильтрацией «грязных» оттенков.
- Формат: `hex`, `name_ru`, `rarity_level` (common / rare / exotic).
- Редкие исторические названия применяются не каждую неделю.
- Color story: 2–3 коротких предложения из внутреннего curated-словаря.
- Никакого web scraping.

## Minimal data model
- `day_signals`: нормализованные метрики по дате + `data_quality` flags.
- `weekly_state`: `week_id` + color data + cache weekly summary.
- `feedback_votes`: date + week_id + vote + timestamp.

## Telegram UX
- Команды: `/today`, `/week`, `/color` (или совместимые эквиваленты).
- Inline-кнопки:
  - Голос: yes / partial / no (✅ / 🤷 / ❌)
  - Показать color story
- Сообщения короткие, чисто форматированные.

## Workflow (incremental)
1. Прочитать текущую архитектуру репозитория.
2. Обновить документацию:
   - `docs/BLUEPRINT_v1.md`
   - `AGENTS.md` (сводка проектных правил)
3. Реализация небольшими шагами:
   - A: color engine + persistence
   - B: procedural visual engine + caching
   - C: daily card composer + fallback
   - D: votes + storage
   - E: weekly report
4. После каждого шага:
   - прогнать тесты/линтеры (если есть),
   - кратко зафиксировать изменения,
   - убедиться, что бот продолжает запускаться.

## Deliverables
- Рабочий код в репозитории с прозрачной структурой.
- Минимальная, но достаточная документация для сохранения контекста.
- Отсутствие требований к платным API.
