# PROJECT_RULES — v1 кратко

1. Весь пользовательский output — на русском языке по умолчанию.
2. Никакого медицинского тона: без диагнозов и предупреждений.
3. Тон бота: «ритм/режим», нейтрально и лаконично.
4. Минимализм обязателен: короткие тексты, без перегруза эмодзи.
5. Юмор редкий и сухой: максимум 2 раза в неделю.
6. В fallback-днях (нет данных) юмор отключён.
7. Визуалы только процедурные (PNG 1080x1080), без scraping и image-каталогов.
8. Недельный цвет фиксируется на ISO-неделю: hex + name_ru + rarity_level.
9. Color story короткая (2–3 предложения) и только из внутреннего словаря.
10. Логика детерминированная, зависимости минимальные.
11. Система устойчива к частичным/пропущенным Garmin-полям.
12. Нельзя ломать существующие рабочие сценарии.
13. Обязательны fallback-ветки, логирование и кэш визуалов.
14. Weekly report ограничен 4–6 ключевыми метриками.
15. DoD v1: /today, /week, /color, feedback ✅/🤷/❌ с хранением и недельной статистикой, без платных API.
