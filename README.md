# Garmin Visual Insight Bot

Telegram-бот, который переводит данные Garmin в спокойные и понятные сигналы про **ритм дня и недели**.  
Фокус: короткие практичные выводы, без медицинского тона, без ложной точности.

## Для кого и зачем

Бот подходит тем, кто хочет:
- видеть свой режим дня (утро / день / вечер) без перегруза графиками;
- получать честный weekly-итог с контрастами и неполнотой данных;
- проверять гипотезы через микро-квест недели.

## Что умеет бот сейчас

- **Daily flow**
  - `/today`: карточка дня (PNG 1080×1080) + краткий статус + факт дня + голосование `✅ / ➖ / ❌`.
  - Scheduled push по слотам `morning / midday / evening` с fallback при неполных данных.
  - Все slot-решения считаются в фиксированном часовом поясе `GMT+3 (Europe/Moscow)` и не подлежат изменению.
- **Color flow**
  - `/color`: цвет недели (детерминирован по ISO-неделе), подпись и история цвета.
  - Голосование по цвету и недельная статистика попаданий.
- **Premium Weekly Card**
  - `/week`: weekly-карточка (изображение + короткий caption).
  - 7-точечная week line (Пн–Вс) со статусами дней.
  - Human-readable чипы вместо сухих KPI.
  - Отдельный блок `🎯 Квест недели` на основе последних дней.
  - Кнопки: **Мой паттерн**, **Что улучшить**, **История цвета**.
- **Refresh flow**
  - `/refresh`: заново подтягивает доступные данные Garmin и честно сообщает, что реально обновилось.
- `/stats`, `/help`.

## Команды

- `/today` — карточка дня.
- `/color` — цвет и история цвета.
- `/week` — Premium Weekly Card.
- `/stats` — статистика голосов недели.
- `/refresh` — обновить доступные данные из Garmin.
- `/help` — список команд.

## Какие данные используются

### Базовые (must-have в логике)
- сон,
- body battery,
- stress,
- resting HR,
- шаги,
- heart rate,
- дневная активность,
- last sync time,
- флаг полноты/неполноты дня (`data_completeness`, `missing_flags`).

### Расширенные (если доступны устройством/интеграцией)
- respiration,
- pulse ox,
- HRV / HRV status,
- intensity minutes,
- calories,
- floors climbed,
- activity summary.

> Важно: часть метрик зависит от модели часов, разрешений и фактической синхронизации Garmin Connect.

## Что значит «неполная картина дня»

День считается неполным, если часть ключевых сигналов не пришла (например, есть stress, но нет sleep/steps). Garmin может прислать их позже, при следующей синхронизации.  
В этом случае бот:
- снижает уверенность формулировок;
- в weekly показывает такие дни как серые/контурные;
- не делает агрессивных выводов.

## Как работает `/refresh`

`/refresh` делает 4 шага:
1. Запрашивает свежие **доступные** данные из Garmin Connect.
2. Делает merge со снимком дня: новые поля дописываются, пустые/`None` не стирают уже сохранённые метрики.
3. После merge заново считает `missing_flags`, `data_completeness`, `confidence`.
4. Возвращает честный ответ: какие блоки появились/обновились и какие сигналы ещё не пришли.

Что `/refresh` **не гарантирует**:
- не может «вытянуть из воздуха» данные, которых ещё нет в Garmin;
- не может принудительно заставить часы дослать сигналы мгновенно;
- если Garmin ещё не синхронизировал сон/HRV/пульс и т.д., бот честно покажет «новых данных нет» и укажет, какие блоки всё ещё отсутствуют.

## Weekly logic (честная)

1. Берутся последние 7 дней.
2. Для каждого дня считается day status:
   - `best` (зелёный),
   - `moderate` (жёлтый),
   - `tense` (красный),
   - `partial` (серый/контур).
3. Считаются:
   - герой-статус недели,
   - цветовой отклик,
   - число дней с неполной картиной,
   - период, который чаще был в ресурсе,
   - квест недели.
4. Отправляется image-card + caption + кнопки.

Если данных мало, weekly использует мягкий статус **«Неделя предварительная»**.

## Ограничения

- Бот зависит от Garmin Connect и качества синхронизации.
- Некоторые сигналы приходят с задержкой (особенно вечером/ночью).
- Не все метрики доступны на всех устройствах.
- Это продукт про ритм/режим, а не медицинский сервис и не медицинский совет.

## Технический запуск

1) Установка:
```bash
pip install -r requirements.txt
```

2) Базовый sync:
```bash
python main.py sync
```

3) Запуск webhook-сервера:
```bash
python main.py serve
```

4) Настройка Telegram webhook на `POST /webhook`.

### Обязательные env/secrets

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `GARMIN_EMAIL`
- `GARMIN_PASSWORD`
- `GEMINI_API_KEY`
- `GEMINI_MODEL`
- `CACHE_GIST_ID`
- `GIST_TOKEN` (или совместимый токен для Gist)

### Проверка push / weekly / refresh

```bash
python -m unittest
python main.py push-self-check
python main.py cache-self-check
PUSH_NOW_MSK=2026-03-01T09:30:00+03:00 python main.py push scheduled
```

## Preview weekly card

```bash
python main.py push scheduled --slot evening
```
(в воскресенье после вечернего слота бот отправит weekly-card автоматически, если не отправлял ранее за неделю).

## Структура проекта

- `main.py` — orchestration: sync, push, webhook, команды Telegram, weekly logic, refresh.
- `cache.py` — хранилище состояния, trim/prune, голосования, weekly state, refresh state.
- `color_engine.py` — детерминированные цвета и генерация изображений (daily/color/weekly).
- `tests/` — unit-тесты сообщений, weekly/refesh логики.
- `docs/` — продуктовые и инженерные заметки.
